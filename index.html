Persistent memory for AI agents

Cognitive AI Memory uses the Ebbinghaus forgetting curve to score, decay, and reinforce memories over time — so your AI always surfaces what matters most, not just what was stored last.

How it works

Memory that ages gracefully

Not a vector database. Not a chat log. A cognitive memory layer that knows what to keep and what to let go.

Ebbinghaus Decay

Memories fade over time using the forgetting curve. Recalled memories get reinforced. Unused ones quietly expire.

Fully Local

SQLite by default — no Docker, no database setup. Upgrade to PostgreSQL + pgvector any time. Your data never leaves your machine.

MCP Native

Works out of the box with Claude Code and any MCP-compatible agent. Three tools: store, recall, update.

Scoring formula

Relevance × recency

Every retrieval combines semantic similarity with a biological memory model. Memories that haven't been accessed in a long time receive a lower strength score — and are pruned entirely once they fall below the threshold.

  • → importance — set at store time (0–1)
  • → λ_eff — decay rate, modulated by importance
  • → recall_count — reinforced on every access
  • → memories below strength 0.05 are pruned
# retrieval score
score = cosine_sim × strength

# ebbinghaus strength
strength = importance
        × exp(−λ_eff × days)
        × (1 + recall_count × 0.2)

# adaptive decay rate
λ_eff = 0.16 × (1 − importance × 0.8)
Setup

Up and running in two steps

No Docker. No database setup. SQLite works out of the box.

1. Install
pip install yourmemory

All dependencies installed automatically. Database created at ~/.yourmemory/memories.db on first run.

2. Wire into Claude (~/.claude/settings.json)
{
  "mcpServers": {
    "yourmemory": {
      "command": "yourmemory"
    }
  }
}

Then reload Claude Code: Cmd+Shift+P → Developer: Reload Window

3. Add memory instructions to your project

Copy sample_CLAUDE.md from the repo into your project root as CLAUDE.md. Replace YOUR_NAME and YOUR_USER_ID. Claude will now recall, store, and update memories automatically on every task.

1. Install PostgreSQL + pgvector

macOS

brew install postgresql@16 pgvector && brew services start postgresql@16

Ubuntu / Debian

sudo apt install postgresql postgresql-contrib postgresql-16-pgvector
2. Create the database
createdb yourmemory
3. Set DATABASE_URL in .env
DATABASE_URL=postgresql://YOUR_USER@localhost:5432/yourmemory

Replace YOUR_USER with your system username (whoami). The backend switches automatically.

4. Install & wire into Claude
pip install yourmemory
{
  "mcpServers": {
    "yourmemory": {
      "command": "yourmemory"
    }
  }
}
